## Project Environment
- Always run in venv311 environment

## File Organization Standards
- **Always follow the established project directory structure** when creating new files or folders
- **Standard directory conventions:**
  - Source code → `src/`
  - Tests → `tests/`
  - Documentation → `docs/`
  - Configuration → `config/`
  - Scripts → `scripts/`
  - Data files → `data/`
  - Test output → `test_output/`
  - Reports → `reports/`
  - Logs → `logs/`
  - Cache → `cache/`
  - Exports → `exports/`
  - Static files → `static/`
  - Examples → `examples/`
  - Assets → `assets/`
- **Never create files in the root directory** unless they are essential project files (README, LICENSE, requirements.txt, etc.)
- **Use appropriate subdirectories** within main directories when organizing related files
- **Follow naming conventions** consistent with existing project structure
